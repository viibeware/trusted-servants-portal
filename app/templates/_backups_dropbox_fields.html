{# Dropbox connection fields — shared by the create wizard's step 2 and the edit page. Implements the OAuth-with-refresh-token flow: 1. Operator creates a Dropbox app (or reuses an existing one) and pastes its App key + App secret. 2. Clicks the auto-generated authorization link, which opens Dropbox's consent screen for THAT app with `token_access_type= offline` so Dropbox issues a long-lived refresh token instead of a short-lived access token. 3. Dropbox shows a one-time authorization code; operator pastes it here. 4. On Save, the server POSTs to https://api.dropboxapi.com/oauth2/ token to exchange the code for a refresh token (one-shot — the code itself is single-use) and stores the refresh token encrypted alongside the app credentials. The SDK then mints a fresh short-lived access token on every API call using the stored credentials — no more 4-hour expiry pain. `t` is the BackupTarget row (template context provides it from either wizard step 2 or the edit page). #}
  1. Visit Dropbox developers → My apps and create a new app (Scoped access, App folder), or open an existing one. Dev mode is fine — the app doesn't need to be published.
  2. Under the new app's Permissions tab, enable files.content.write, files.content.read, files.metadata.write, files.metadata.read. Save.
  3. Open the Settings tab and copy the App key and App secret into the two fields below.
  4. Click Open Dropbox authorization page. Dropbox will ask you to grant the app access; after you allow, it shows a one-time authorization code. Paste that code into the third field.
