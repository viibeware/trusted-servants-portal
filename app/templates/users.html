Roles & permissions

What each role can access. Module access is also gated per-module on the Modules tab.
  • Admin
    Full access. Manages users, modules, settings, the Web Frontend, and every other feature in the portal.
  • Editor
    Edits existing meetings, locations, schedules, Zoom accounts, and meeting attachments (any uploader). Cannot create new meetings or delete existing ones — that's gated to admins and Intergroup Members. On libraries: may add files to existing libraries (and edit / delete only those whose uploader was another Editor — admin-, Intergroup-Member-, and legacy library files are protected); cannot create new libraries or edit library settings (name, description, alert) — admin / Intergroup Member only. Cannot reach Settings, Users, or the Web Frontend module.
  • Intergroup member
    Inherits every Editor permission (meetings, libraries, readings, uploads), plus exclusive edit access to Intergroup-flagged libraries — which regular Editors cannot touch. May delete any library file. Cannot reach Settings, Users, the Web Frontend module, or the Intergroup Email Accounts page (admin-only).
  • Viewer
    Read-only. Sees meetings, libraries, files, and any module the admin opens to All signed-in users; cannot edit anything.
